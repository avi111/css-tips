## 🔴 הסיפור המרכזי: המהפכה הושלמה - עברנו ל-Vite!

**כתבנו לענייני ארכיטקטורה מדווח:**

היום אנו מדווחים על שינוי דרמטי בארכיטקטורת הפרויקט `css-tips`.
מערכת הבנייה הישנה הוחלפה ב-**Vite** המודרנית. המשמעות היא שהדפדפן שלכם מריץ כעת את קוד ה-TypeScript כ-ES Modules טבעיים בסביבת הפיתוח.

**מה זה אומר עבור המפתחים?**
*   ⚡ **מהירות:** זמני טעינה אפסיים.
*   🔄 **רענון:** רענון מיידי של השינויים (HMR).
*   🛠 **פשטות:** תמיכה מובנית ב-TypeScript ללא קונפיגורציה מסובכת.

*(דיווח זה מופיע גם בעמוד הראשי וגם כאן כחלק מההיילייטס של המהדורה)*

## ✨ חידושי ה-CSS: קינון (Nesting) ומשתנים הגיעו אלינו!

**כתבנו לענייני עיצוב מדווח:**

כחלק מהשדרוג האחרון, קובץ ה-`index.css` עבר מתיחת פנים מודרנית. השינויים העיקריים:

1.  **CSS Nesting**: כבר לא צריך להשתמש ב-Sass או Less כדי לקנן סלקטורים. אנו משתמשים בתחביר הקינון המודרני הנתמך בדפדפנים:
    ```css
    .card {
      /* ... */
      & h1 { color: var(--accent-color); }
      &:hover { transform: translateY(-2px); }
    }
    ```

2.  **משתני CSS**: הוגדרו משתנים גלובליים (`:root`) לצבעים ופונטים, מה שמאפשר שליטה מרכזית על העיצוב.

## 🛠 דיווח טכני: סביבת הפיתוח

בגזרת הפיתוח נרשמה התייעלות משמעותית.
כאשר אתם מריצים את הפקודה:
```bash
npm run dev
```
השרת של Vite מופעל. קובץ ה-`index.html` טוען ישירות את `/src/index.ts` באמצעות תגיית `<script type="module">`.
המשמעות: אין צורך בבנדלינג (Bundling) איטי בכל שמירה של קובץ.

<a id="prod-build"></a>
## 🚀 דיווח טכני: מוכנות לפרודקשן

כתבנו לענייני DevOps מעדכן כי סביבת הייצור (Production) עברה שדרוג.
בהרצת הפקודה:
```bash
npm run build
```
מתבצע תהליך דו-שלבי:
1.  **בדיקת טיפוסים:** `tsc` מוודא שאין שגיאות TypeScript.
2.  **בנייה:** `vite build` יוצר באנדל אופטימלי, מכווץ (Minified) ומוכן להפצה בתיקיית `dist`.
